CSS Scroll-Driven Animation

  • Instead of using JavaScript timers (setTimeout and setInterval), we are going to use CSS Scroll-Driven Animation in this case. There are various use cases for this, for example, ads display window and "go back to top" button, and so on. When you are scrolling, this component will hide itself; while you quit scrolling, it will appear. The case not only detects the scroll event, but it also can detect the scroll speed and direction.
  • REMEMBER: there are many useless code which its absence would not affect the funciton of this code. I am writing it out for checking the scroll event. I used --scroll-velocity to see the time gap (which is also the difference between --scroll-position and --scroll-position-delay), to represent the scroll status. 0 represents the QUIT status, not moving at all. Positive figure represents scrolling down. Negative figure represents scrolling up.
  • WHY --scroll-velocity is important? We are using it to represents the scroll event status, and according to various status, we would define various CSS for them. In this case, when we are scrolling, the component would hide itself. While we quit scrolling, the component will appear. Therefore, we need only two status, SCROLLING and QUIT, which are ANY FIGURE and ZERO, respectively.

See the Pen CSS Scroll-Driven Animation by Jiaying Yu (@JiayingYu152) on CodePen.
